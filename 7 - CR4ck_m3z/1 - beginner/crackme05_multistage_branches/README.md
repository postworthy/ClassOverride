# Crackme 05: Multistage Branches

## Difficulty
Beginner

## Challenge Intent
Build a short crackme with several validation stages and distinct failure messages to teach branch-oriented debugging.

## Student Goal
Pass every stage and reach the final success branch.

## Learning Objectives
- Use branch outcomes to narrow down correct input
- Observe how failure messages reveal progress
- Practice stepping through a multi-check validator

## Planned Implementation Notes
- Three to five ordered checks
- Different failure text for each stage
- Final input still human-readable and deterministic

## Intended Solve Path
Students should use the messages and branch behavior to understand which stage fails and why.

## Constraints
- Keep each stage conceptually simple
- Avoid misleading dead code at this level
