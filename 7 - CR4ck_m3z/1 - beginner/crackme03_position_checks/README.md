# Crackme 03: Position Checks

## Difficulty
Beginner

## Challenge Intent
Validate user input through a series of character-by-character checks instead of a full string comparison.

## Student Goal
Determine the valid input by analyzing branch conditions.

## Learning Objectives
- Follow conditional logic in a small function
- Map character positions to comparisons
- Use debugger stepping to confirm assumptions

## Planned Implementation Notes
- Enforce exact length
- Check a handful of positions directly
- Use simple arithmetic relationships for one or two characters

## Intended Solve Path
Students should trace the checks and derive the answer one constraint at a time.

## Constraints
- Small enough to solve in one sitting
- No hidden inputs or indirect control flow
