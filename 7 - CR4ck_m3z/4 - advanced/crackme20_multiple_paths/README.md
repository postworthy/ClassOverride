# Crackme 20: Multiple Paths

## Difficulty
Advanced

## Challenge Intent
Provide more than one valid route to success, including at least one hidden or backdoor-style path, to reward thorough analysis.

## Student Goal
Find one or all success conditions and explain how they differ.

## Learning Objectives
- Avoid stopping after the first apparent answer
- Compare overt validation logic with hidden alternate logic
- Practice complete program exploration

## Planned Implementation Notes
- Main path with normal validation
- Secondary hidden path triggered by a separate condition
- Instructor notes should document both intended discoveries

## Intended Solve Path
Students should analyze the entire binary and identify all meaningful success paths, not just the first one encountered.

## Constraints
- Hidden path must be discoverable from the binary itself
- Keep alternate paths deliberate and ethically safe
